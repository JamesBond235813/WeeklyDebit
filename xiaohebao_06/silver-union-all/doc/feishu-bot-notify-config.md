# 飞书机器人上游客户通知配置

## 功能范围

系统收到上游 API 推送客户，并且客户新增入库成功后，会向飞书群机器人发送文本通知。

不发送通知的情况：

- 手工新增客户
- Excel 导入客户
- access-check 撞库拒绝
- 未先调用 access-check
- request_id 重复
- 手机号重复
- 解密失败
- 参数错误
- 未新增入库的重复客户

## 配置项

后端读取配置项：

```yaml
biz:
  feishuBotWebhook: ''
```

为空时不发送飞书通知。

云端生产环境建议通过 systemd 启动参数注入：

```bash
--biz.feishuBotWebhook=<飞书机器人 webhook>
```

说明：本说明文件只保留配置项名称和使用方式，不保存 webhook 明文。

## 消息去向口径

- `ownerUserId` 为空或小于等于 0：提示 `进入公海中`
- `ownerUserId` 大于 0：查询后台用户真实姓名，提示 `分配至"用户名"`

飞书发送失败只记录日志，不影响客户入库，也不影响上游接口返回。
