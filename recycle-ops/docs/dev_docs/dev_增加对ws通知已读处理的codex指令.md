# 对ws通知增加已读处理

## 角色定义
你是一个经验丰富的高级全栈开发工程师，精通java, vue3.5 以及antd等各种前端框架，目标是生成稳定、可维护、最小风险的生产级代码。

## 1. 目标定义（Objective）
未读消息列表中,点击`立即处理`后， 页面按当前已有逻辑跳转，且该消息被标记为已读。


## 2. 完成标准（Definition of Done）
1. 页面 layout 右上角,点击`查看全部未读`， 在弹出的未读消息列表中：
   - 点击`立即处理`后， 页面按当前已有逻辑跳转，且该消息被标记为已读，不出现在未读消息列表中。
   - 新消息图标旁边的未读消息数字减1，若未消息数字小于等于0，则不显示未读消息数字。
   - 刷新页面后，已读的消息不会被纳入新消息计数，且不出现在未读消息列表中。

2. 若当前没有未读消息，则不出现`您有新数据， 请及时处理 查看全部未读` 的提示。
   
3. 在未登录的情况下， websocket请求服务端， 服务端应返回 http 401状态码，且前端不再重试立websocket连接，直到用户登录后再请求建立websocket连接。
